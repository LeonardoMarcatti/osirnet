FIREWALL
	1 - Para manter as configurações de firewall salvas mesmo após dar reboot no server basta instalar o pacote iptables-persistent e fazer as configurações na tela.
		1.1 Após fazer alterações usar o comando iptables-save > /etc/iptables/rules.v4 para salva-las.
		
	2 - Liberar apt-get com regras padrão de firewall em DROP com SQUID ativo. 
		2.1 Editar o aquivo na máquina cliente /etc/bash.bashrc e adicionar no final os campos:
		export http_proxy="http://ip_rede_interna_server:3128" e export https_proxy="http://ip_rede_interna_server:3128" 
	
ENCAMINHAMENTO E MASCARAMENTO DE PACOTES
	1 - Editar o arquivo  /etc/syscrtl.conf e descomentar a linha net.ipv4.ip_forward = 1.
	2 - iptables -t nat -A POSTROUTING -o ethx -j MASQUERADE
	
ADICIONAR ROTAS ESTÁTICAS
	1 - vim /etc/network/interfaces e adicionar o comando após a última linha na interface eth0
			post-up route add -net ip_da_rede_esejado/24 gw ip da máquina que conhece o ip final desejado
			
SUBSTITUIR PALAVRAS
			
:%s/texto_velho/texto_novo/g

ENCONTRAR ARQUIVOS QUE POSSUAM STRINGS EXPECÍFICAS

find / -type f -exec grep -l TESTE {} \; 

TECLADO NUMÉRICO NO VIM VIA SSH
Adicionar ao final do arquivo /etc/vim/vimrc os  seguintes códigos:
echo 'imap <Esc>Oq 1
imap <Esc>Or 2
imap <Esc>Os 3
imap <Esc>Ot 4
imap <Esc>Ou 5
imap <Esc>Ov 6
imap <Esc>Ow 7
imap <Esc>Ox 8
imap <Esc>Oy 9
imap <Esc>Op 0
imap <Esc>On .
imap <Esc>OQ /
imap <Esc>OR *
imap <Esc>Ol +
imap <Esc>OS -
set number' >> /etc/vim/vimrc
Salvar e sair.
